## 2024-05-24 - [Critical XSS in HTML Formatter via Cheerio Testing Pitfall]
**Vulnerability:** XSS vulnerability in the HTML CLI Formatter (`lib/cli-engine/formatters/html.js`) where `ruleId` and `ruleUrl` variables derived from external configuration metadata were not HTML-encoded before being injected into HTML template strings.
**Learning:** This vulnerability existed unchecked for potentially a long time because the unit testing approach using Cheerio was fundamentally flawed for verifying raw encoded entities. Cheerio parses HTML attributes and automatically unescapes numerical entities, meaning test assertions against Cheerio's parsed output completely masked the fact that the underlying raw HTML string was unescaped and vulnerable.
**Prevention:** When testing code that outputs raw strings intended to contain HTML entities (especially formatters or templating logic), ALWAYS assert on the raw string output directly using string matching (e.g., `result.includes('&#60;')`) rather than relying on an HTML parser like Cheerio which may silently alter the payload and give a false sense of security.
